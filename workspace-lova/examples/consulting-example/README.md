# Consulting Project Example

This example demonstrates the folder structure for a consulting or client services project.

## Project Overview
- **Client**: Acme Corporation
- **Project**: Digital Transformation Strategy
- **Duration**: 3 months
- **Modules used**: Core structure + Consulting module

## Structure Explanation

### `/client/`
- `brief/`: Client requirements, project scope, objectives
  - `initial-brief.md`
  - `scope-of-work.pdf`
  - `success-metrics.md`
- `deliverables/`: Client deliverables (reports, code, presentations)
  - `phase-1/`
  - `phase-2/`
  - `final-delivery/`
- `communications/`: Meeting notes, email correspondence, status updates
  - `meeting-notes/`
  - `email-threads/`
  - `status-reports/`
- `invoices/`: Invoices, payment records, expense tracking
  - `invoice-001.pdf`
  - `payment-confirmations/`
  - `expense-reports/`

### `/research/`
- `competitive-analysis/`: Competitor research, market analysis
- `interviews/`: Stakeholder interview transcripts, insights
- `findings/`: Research findings, data analysis, insights

### `/presentations/`
- `proposals/`: Initial proposal, pitch deck
- `reports/`: Progress reports, final report
- `workshops/`: Workshop materials, exercises, handouts

## Project Workflow

### Phase 1: Discovery
1. Client briefing documents in `/client/brief/`
2. Initial research in `/research/`
3. Proposal development in `/presentations/proposals/`

### Phase 2: Execution
1. Regular communications in `/client/communications/`
2. Research findings in `/research/findings/`
3. Progress reports in `/presentations/reports/`

### Phase 3: Delivery
1. Final deliverables in `/client/deliverables/final/`
2. Final presentation in `/presentations/reports/final/`
3. Invoicing in `/client/invoices/`

## Client Management Tips

### Documentation
- Keep all client communications organized
- Document decisions and rationale
- Version control for deliverables

### Invoicing
- Consistent invoice numbering
- Track payment status
- Record expenses against project budget

### Deliverables
- Clear naming conventions (YYYY-MM-DD-Description)
- Include both source files and final versions
- Add README files explaining deliverables

## Tools Integration

- **Project management**: Asana/Trello integration (link in docs)
- **Time tracking**: Toggl/Harvest integration
- **Document collaboration**: Google Drive/Notion links
- **Communication**: Slack/Teams channel archives
