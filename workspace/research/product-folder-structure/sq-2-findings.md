# Sub-Question 2 Findings: Digital Product Assets

**Source 1:** ImageKit - Marketing Folder Structure
**URL:** https://imagekit.io/blog/marketing-folder-structure/

## Key Findings

### Marketing Folder Structure Example
- Main folder: "Marketing Materials"
- Subfolders per marketing campaign
- Campaign folders contain: `assets/`, `copy/`, `design/`, `social/`, `analytics/`
- Assets organized by type: `images/`, `videos/`, `documents/`
- Version control through `drafts/` and `final/` subfolders

### Digital Asset Organization (EvergreenFeed)
**URL:** https://www.evergreenfeed.com/blog/how-to-organize-digital-assets/
**Accessed:** 2026-03-19

- Department-based organization: `Marketing/`, `Sales/`, `Product/`
- Within Marketing: `Social Media/`, `Blog Content/`, `Campaigns/`
- Product-based organization: `Product A/`, `Product B/`
- Asset type organization: `Images/`, `Videos/`, `Documents/`, `Design Files/`

### DAM Folder Structure (Pics.io)
**URL:** https://blog.pics.io/how-to-create-a-folder-structure-for-a-digital-asset-management-dam-system/
**Accessed:** 2026-03-19

- Start with asset types: images, videos, audio files, documents, design files
- Consider roles and permissions: Marketing vs Design vs Product teams
- Balance folders with metadata for searchability
- Avoid overly deep nesting (max 3-4 levels)

### Marketing Materials Organization (GainApp)
**URL:** https://blog.gainapp.com/how-to-organize-marketing-materials/
**Accessed:** 2026-03-19

- Creative agency folder structure:
  - `Client Assets/`
  - `Internal Assets/`
  - `Templates/`
- Within each: break down by project or asset type, then date or version
- Use consistent naming conventions

### Organizing Digital Assets (Canto)
**URL:** https://www.canto.com/blog/organizing-digital-assets/
**Accessed:** 2026-03-19

- Folder categorization schema by product line, campaign type, content type
- Assets should be searchable beyond just folder structure
- Use metadata and tags for cross-cutting organization

## Patterns Identified

1. **Multiple organizational dimensions**: Department, Product, Asset Type, Campaign
2. **Consistent hierarchy**: Top-level categories with logical subfolders
3. **Version control**: `drafts/`, `final/`, `archive/` folders
4. **Permission separation**: Different teams need different access
5. **Metadata complement**: Folders alone insufficient; need searchable metadata

## Implications for Digital Product Structure
- Need separate `assets/` or `marketing/` folder for product-specific assets
- Should support multiple organizational schemes
- Include versioning and approval workflows
- Separate source files from distribution files (e.g., `source/` vs `export/`)
- Consider product lifecycle: `concept/`, `development/`, `launch/`, `maintenance/`
