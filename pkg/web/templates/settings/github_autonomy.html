GitHub Autonomy

Enable poller

Periodically scans the listed repositories for open issues labeled below and runs the fix-github-issue workflow for each new one. Push + PR-create always require approval. Requires gh CLI to be authenticated and an LLM-backed Fixer to be wired — without one, the workflow halts at the fix step.

The poller is a no-op when this list is empty, even if Enabled is on.

Issues without this label are ignored.

Each issue is cloned into <root>/<owner>-<repo>-<n>.

Push via fork

When on, adapters push to a user fork instead of the upstream origin. Useful when you don't have write access — the PR becomes a cross-repo PR.

Poller is enabled but no repositories are configured — nothing will be scanned.
How it works: the poller lists open issues matching the label, skips ones already processed (stored in the semantic memory graph), and spawns the fix-github-issue workflow per new issue. The workflow clones, reproduces, attempts a fix, and stops at push — where the approval queue in the Monitor tab takes over.
